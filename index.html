How Code Was plagiarize ?

In code plagiarism detection, two or more sets of code are compared to identify similarities
or instances of copied code. Several aspects are typically checked when comparing
two code snippets or programs to detect plagiarism:

  • Code Structure and Logic

    • Plagiarism detection tools analyze the overall structure and logic of the code to identify similarities.
    This includes checking for similarities in the flow of control structures like loops and conditionals,
    function or method structures, and the organization of code blocks.

  • Code Tokens

    • Code is broken down into smaller units called tokens, which can be keywords, operators, identifiers,
    literals, or other code elements. Plagiarism detection tools compare these tokens to
    identify common sequences or patterns.

  • Variable and Function Names

    • While the actual variable and function names might differ, plagiarism detection can still identify
    similarities by comparing the usage and pattern of these names. Renaming variables or
    functions is a common tactic used by plagiarizers.

  • Comments and Documentation

    • The presence of similar comments or documentation within the code can also be used as a clue
    for plagiarism detection. This includes checking for identical or paraphrased comments.

  • tring Literals and Constants

    • Plagiarism detection can identify similarities in string literals and constants used in the code.
    Even if variable names and structure differ, matching string literals can indicate potential plagiarism.

  • Whitespace and Formatting

    • Code formatting, including the use of whitespace and indentation, can also be checked for similarities.
    Some plagiarism detection tools normalize code formatting to make comparisons more effective.

    It's important to note that not all code similarities are indicative of plagiarism. Developers often encounter similar problems and use common coding patterns, resulting in unintentional code similarities. Plagiarism detection tools aim to identify cases of deliberate code copying or excessive reuse. Additionally, the specific techniques and algorithms used for code plagiarism detection may vary between different tools and systems, but the principles mentioned above are commonly employed in such detection processes.
